# Task 986 — Replacement checkpoint after cancelling stuck tasks

## 2 lỗ hổng practical nên đẩy tiếp ngay
1. **Public route parity chưa chốt sạch** — sau khi thay batch cho các task treo, nên ưu tiên route public còn lệch nhất (top-picks / guides / story detail) để tất cả public slice nhìn đồng đều hơn, không còn chỗ “được chăm” nhiều khác nhau.
2. **Admin operator feedback chưa đồng đều** — sau public, nên đẩy một batch rất hẹp cho admin save/help/status cues để các route admin quan trọng (homepage/navigation/promos) có mức hướng dẫn/feedback gần nhau hơn.
